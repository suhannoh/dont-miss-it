# Project Rules

## 프로젝트명

놓치면 알려줘

## 핵심 정의

관심 있는 팝업·굿즈·티켓·행사 공식 페이지를 저장하면, 예약오픈·재입고·품절해제·운영시간 변경·장소 변경·공지 변경처럼 사용자가 행동해야 하는 변화가 생겼을 때 알려주는 알림 서비스다.

## 핵심 방향

이 프로젝트는 추천 서비스가 아니다.

이 프로젝트는 사용자가 이미 관심을 가진 공식 페이지의 상태 변화를 감시하고, 행동 가능한 변화가 발생했을 때 알림을 보내는 운영형 백엔드 시스템이다.

## 절대 하지 말아야 할 것

- 팝업 목록 서비스로 만들지 않는다.
- 티켓팅 일정 통합 서비스로 만들지 않는다.
- 굿즈 쇼핑몰 통합 검색 서비스로 만들지 않는다.
- 인기 랭킹, 리뷰, 커뮤니티를 만들지 않는다.
- 자동 구매, 자동 예약, 매크로 기능을 만들지 않는다.
- 인스타그램, X, 네이버 예약 등 동적/폐쇄 플랫폼을 무리하게 크롤링하지 않는다.
- AI 추천 서비스를 만들지 않는다.
- 범용 URL 변경 감지 도구처럼 보이게 만들지 않는다.

## 반드시 집중할 것

- URL 등록
- HTML Fetch
- Text Extraction
- PageSnapshot 저장
- Snapshot Diff
- ChangeType 감지
- ChangeEvent 생성
- Redis Lock
- 중복 알림 방지
- Fetch / Parse 실패 관리
- Source Health
- Notification Log
- 관리자 운영 대시보드

## 최종 기준

추천 / 목록 / 랭킹으로 가지 않고, 감시 / 변경 / 알림 / 운영에 집중한다.
